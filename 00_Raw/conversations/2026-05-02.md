## 🧩 Godot 기반 '능동적 자아 재구성' 유도 핵심 메커니즘 도출 (3가지)

**목표 분석:** 단순한 엔터테인먼트를 넘어선 '능동적 행위자로서의 자아 재구성'을 경험하게 하려면, 플레이어의 **과거 기록(데이터)**이 현재의 **판단(선택)**에 영향을 미치고, 그 결과가 다시 **미래의 나 자신**에게 반영되어야 합니다.

아래 3가지 메커니즘은 Godot 엔진의 노드 기반 시스템 및 신호(Signal) 연결 구조를 최대한 활용하여 설계되었습니다.

---

### 1. 메커니즘: 에코 로그 (The Echo Log System)
*   **핵심 기능:** 플레이어의 모든 중요한 선택, 상호작용, 실패 경험이 단순한 변수 값이 아닌 **'기록 가능한 데이터 파편(Echo)'**으로 저장됩니다. 이 기록은 게임 내 환경이나 대화에 물리적으로 영향을 미칩니다.
*   **🧠 자아 재구성 관점:** 플레이어는 자신이 무의식적으로 만들어낸 '나만의 역사'를 눈으로 확인합니다. "내가 저때 그렇게 결정했기 때문에, 지금 세상이 이렇게 반응하는구나"라는 인식을 통해 능동적 행위자로서의 책임감을 느낍니다.
*   **⚙️ Godot 구현 포인트:**
    1.  **`EchoResource` (Custom Resource):** 선택 내용(Key), 발생 시점/장소(Context ID), 결과 값(Effect Value)을 담는 전용 리소스 타입을 만듭니다.
    2.  **Signal 연결:** 중요한 상호작용 노드에서 `choice_made(echo_data)` 신호를 방출하고, 이 신호를 **`Global Data Manager (Autoload/Singleton)`**가 수신하여 에코 로그 데이터베이스에 누적합니다.
    3.  **반영:** 특정 장소나 NPC와의 대화 시, `Global Data Manager`는 플레이어의 최근 5개 `EchoResource`를 분석하고, 이 기록과 관련된 **특정 문구(Dialogue Snippet)** 또는 **환경 변화 (Visual Node Change)**를 강제로 불러와 노출합니다.
*   **예시:** [선택] 거짓말을 선택했다 $\rightarrow$ 에코 로그에 '기만' 데이터 추가 $\rightarrow$ 3챕터에서 아무도 믿지 않는 NPC가 나타나며, "당신은 이미 한 번 속였어요."라는 대사(Echo 기반)를 출력합니다.

### 2. 메커니즘: 트레이트 합성 시스템 (Trait Synthesis System)
*   **핵심 기능:** 플레이어의 선택이 스탯 포인트 증가 같은 수치적 보상이 아니라, **'내면적 성향(Trait)'**이라는 비가역적인 특성으로 합성됩니다. 이 트레이트는 행동의 *방식*을 변화시킵니다.
*   **🧠 자아 재구성 관점:** 플레이어는 자신이 '착한 사람', '실용주의자', '지나친 낙관론자'와 같은 고정된 정체성을 갖게 된다고 느낍니다. 이 트레이트는 캐릭터의 **필터(Filter)** 역할을 하여, 세상을 해석하는 방식 자체를 바꿉니다 (예: 낙관적 트레이트 보유 시 부정적인 사건을 '배움의 기회'로 인식하고 대화 옵션이 달라짐).
*   **⚙️ Godot 구현 포인트:**
    1.  **`Trait` Enum/Resource:** 최소 5~7개의 상호 배타적이거나 보완적인 트레이트 리소스를 정의합니다 (예: `Empathy`, `Cynicism`, `Pragmatism`).
    2.  **Weighted Choice Logic:** 대화 선택지(`Option Button`)에 **가중치(Weight)**를 부여하고, 이 가중치는 현재 플레이어의 보유 트레이트에 의해 결정됩니다.
        *   *(수식 예시):* $\text{선택 가능 여부} = \text{Base Difficulty} + (\text{Trait}_{\text{A}} \times \text{Bonus}_{\text{A}}) - (\text{Trait}_{\text{B}} \times \text{Penalty}_{\text{B}})$
    3.  **Godot State Machine:** 트레이트 변화는 단순 변수 업데이트를 넘어, 캐릭터의 **상태 머신(State Machine)** 자체에 영향을 주어 특정 행동을 아예 불가능하게 만들거나(제약), 새로운 액션을 가능하게 합니다 (개방).

### 3. 메커니즘: 결정 웹 (The Decision Web)
*   **핵심 기능:** 모든 스토리 전개가 선형적이지 않고, 중앙의 '갈등 노드'를 중심으로 여러 개의 작은 **결정 브랜치(Micro-Decisions)**가 복합적으로 연결된 거미줄 구조입니다. 이 각 브랜치의 결과는 다른 브랜치에 영향을 미치는 *조건*이 됩니다.
*   **🧠 자아 재구성 관점:** 플레이어는 자신이 직면한 문제가 단일 원인이 아니라, 여러 과거의 선택(웹의 다른 부분)들이 복합적으로 얽혀 발생했음을 깨닫습니다. 문제 해결 과정 자체가 '전체적인 나'를 이해하는 과정이 됩니다.
*   **⚙️ Godot 구현 포인트:**
    1.  **Graph/Node System:** 모든 갈등과 결정 지점을 노드(Node)로 설계합니다. 각 노드는 여러 개의 입력 포트와 출력 포트를 가집니다.
    2.  **Dependency Signal:** 한 브랜치에서 중요한 데이터가 생성되면 (예: '자원 A 획득', '인물 B와의 신뢰도 하락'), 해당 데이터는 연결된 다음 노드(Next Node)의 **필수 조건(Prerequisite)** 역할을 합니다.
    3.  **Conditional Flow:** `Node A`를 거치기 위해서는 반드시 `Echo Log`에서 '용기' 관련 에코가 최소 1개 이상 존재해야 하는 등의 복합적인 조건을 Godot 스크립트(`_process` 또는 신호 리스너)로 구현합니다.

### 📜 핵심 코드 산출물

#### 1. `IModuleInterface.gd` (가상 인터페이스/계약서 정의)

이 스크립트는 모든 콘텐츠 모듈이 반드시 가져야 할 메서드 목록을 정의합니다. 실제 GDScript는 강력한 인터페이스를 지원하지 않으므로, 이 코드를 **모든 모듈의 부모 클래스**로 사용하고 `_init()` 함수에서 강제 로딩 및 체크를 수행하는 것이 가장 안전합니다.

```gdscript
# scripts/IModuleInterface.gd (Abstract Base Class 역할)
class_name IModuleInterface extends Node
extends Node

@export var module_name: String = "Unnamed Module" # 모듈 식별자 필수

# 이 함수는 모든 콘텐츠가 시작될 때 가장 먼저 호출되는 초기화 지점입니다.
# 자식 클래스(실제 모듈)는 반드시 오버라이드 해야 합니다.
func initialize(global_context: Dictionary):
    print("--- [{}]: 초기화 시작 ---".format([module_name]))
    if not global_context.has("player_data"):
        push_error("IModuleInterface 오류: Global Context에 'player_data'가 누락되었습니다.")
        return false

# 모듈이 특정 이벤트(예: 대화 종료, 아이템 획득)를 처리할 때 호출됩니다.
# 매니저가 이 함수를 통해 콘텐츠 로직을 분기합니다.
func handle_event(event_type: String, data: Dictionary = {}) -> bool:
    print("--- [{}]: 이벤트 수신 - {} ---".format([module_name, event_type]))
    return false # 기본값은 처리를 하지 않음

# 모듈이 게임 루프에서 주기적으로 업데이트해야 할 로직 (선택 사항)
func update(delta: float):
    pass

# 모듈을 메모리에서 안전하게 제거하는 정리 함수
func shutdown():
    print("--- [{}]: 종료 및 리소스 해제 완료 ---".format([module_name]))
```

#### 2. `GameManager.gd` (중앙 제어기 / Singleton)

이 스크립트는 **Autoload**로 설정되어야 하며, 모든 모듈을 관리하고 실행하는 유일한 출입구입니다.

```gdscript
# scripts/GameManager.gd (Singleton으로 등록할 파일)
extends Node

# 로드된 모든 활성 모듈 인스턴스를 저장합니다. (Key: Module Name)
var active_modules: Dictionary = {} 
const MODULE_PATH = "res://modules/" # 모듈 폴더 경로 지정

func _ready():
    print("=============================================")
    print("[GameManager] 시스템 부팅 완료. 모듈 로드 준비.")
    # 시작 시, 필수 모듈들을 강제로 초기화합니다.
    load_all_required_modules(["dialogue_module", "trait_engine_module"])

# -------------------------------------------
# [핵심 API: 모듈 관리]
# -------------------------------------------

# 외부 콘텐츠를 동적으로 로드하고 초기화하는 메인 함수 (API 호출 지점)
func load_all_required_modules(module_names: Array):
    var global_context = {"player_data": {} /* 실제 플레이어 데이터를 주입 */}
    for module_name in module_names:
        var path = MODULE_PATH + "%s.gd" % module_name
        var ModuleScene = load(path) # GDScript를 Resource로 로드 시도

        if ModuleScene is Node and ModuleScene.has_method("initialize"):
            var new_module = ModuleScene.instantiate() as IModuleInterface
            new_module.module_name = module_name
            
            # 1. 초기화 (계약서 강제 실행)
            if new_module.initialize(global_context):
                active_modules[module_name] = new_module
                add_child(new_module) # 씬 트리에 추가하여 메모리 유지

func execute_event(event_type: String, data: Dictionary = {}):
    """
    메인 스토리라인에서 이벤트가 발생했을 때 모든 활성 모듈에 이벤트를 전달합니다.
    각 모듈은 자신에게 필요한 로직만 처리하고 나머지는 무시합니다 (Decoupling).
    """
    print("\n[GameManager] ======================================")
    print("[GameManager] 시스템 이벤트 발생: {}".format([event_type]))
    var successful_handling = false
    for module_name in active_modules.keys():
        var module = active_modules[module_name]
        if module.has_method("handle_event"):
            # 모듈별로 이벤트를 전파하고, 성공 여부를 체크합니다.
            var success = module.handle_event(event_type, data)
            if success:
                successful_handling = true
    
    if not successful_handling:
        print("[GameManager] 경고: 어떤 모듈도 이 이벤트를 처리하지 않았습니다.")

# -------------------------------------------
# [게임 루프 연동]
# -------------------------------------------

func _process(delta):
    for module in active_modules.values():
        if module.has_method("update"):
            module.update(delta)
```
